# 2026-03-19 优化说明

## 目标

- 保持现有功能可用
- 提升桌面即开即用体验
- 提升上传与迁移处理性能
- 补齐模板导入导出
- 将监控页改为真实数据展示

## 改动清单

### 1) 桌面即开即用

- 新增 `backend/desktop_app.py`
  - 后台线程启动 FastAPI
  - 检测后端就绪后以 pywebview 打开本地窗口
- `scripts/start.bat` 改为默认桌面模式
- 新增 `scripts/start_backend.bat` 保留纯后端调试入口
- `scripts/build.bat` 改为打包 `desktop_app.py`

### 2) 动态并发 + 线程安全

- 新增 `backend/utils/system_resource.py`
  - 自动探测 CPU / 可用内存
  - 动态计算 worker 数
- `backend/main.py`
  - `run_migration_task` 重构为并发异步处理
  - 共享状态写入加锁（`task_state_lock`）
  - 迁移进度、文件状态、吞吐图、统计数据实时推送

### 3) 模板导入导出

- 前端 `frontend/index.html`
  - 模板管理区新增“导入 JSON 模板”“导出当前模板”按钮
  - 导入成功后自动刷新、自动定位并打开新模板

### 4) 实时数据展示

- 前端取消默认图表模拟任务注入
- 仅在“运行演示”模式下启动模拟波动
- 正常模式使用 WebSocket 推送与后端统计真实更新

### 5) 稳定性修复

- `backend/core/llm/processor.py`
  - 修复 `double_layer_process` 中 `quality_check` 调用
- `backend/models/template.py`
  - 去除同步场景 `run_until_complete`，改为同步缓存读取

## 验证

- 语法检查：`python -m py_compile`（backend 全量）
- 测试：`pytest tests -q` 通过

## 兼容性

- 未删除原有接口
- 原有任务管理、授权、上传流程保持可用

## 2026-03-20 增量修复

### 1) 导出模板无响应修复

- 后端新增本地导出接口：`POST /api/templates/{template_id}/export/save`
- 前端“导出当前模板”优先调用本地导出接口，在桌面模式下返回落盘路径并提示用户
- 保留 `GET /api/templates/{template_id}/export` 作为浏览器下载兼容路径

### 2) 统一项目内弹窗

- 前端移除原生 `alert/confirm/prompt` 与 `window.open`
- 统一改用内置弹窗：`appAlert` / `appConfirm` / `appPrompt`
- OAuth 授权流程改为当前窗口跳转，回调完成后返回首页并刷新授权状态

### 3) 第二次上传无反应修复

- 新建任务时重置 `#file-input.value`
- 文件选择后重置 `e.target.value`，支持重复选择同一文件
- 启动任务时同步设置 `currentTaskId`，避免前端状态不同步导致进度不刷新

### 4) 重复文件状态闭环

- 上传接口新增知识库同名检测（按文件名/去扩展名归一化比较）
- 命中同名后跳过上传并标记为 `duplicate` 状态
- 后端统计新增 `duplicate` 字段并通过 WebSocket 推送
- 监控面板新增“成功/重复/失败”状态卡片，点击后弹出居中文件明细弹窗（支持滚动）

### 5) 模板与节点编辑体验收口

- 新建任务弹窗在打开前强制刷新模板下拉，确保模板列表实时可选
- 模板创建/导入/编辑/删除后，同步刷新新建任务模板列表
- 节点编辑面板移除“类型选择”，新增/编辑节点统一为 `folder`

### 6) 配置页精简与模型切换修复

- 移除配置页“最低质量分数”输入框
- 移除飞书 API 配置中的“知识空间 ID”输入框
- `MinerU` 配置区块前端暂时隐藏（保留后端能力）
- 修复 LLM provider 切换时 API Key 状态显示异常：
  - 后端 `/api/config` 返回完整 `has_api_key_map`
  - 前端按 provider 实时更新占位提示
  - 保存时保留未修改 provider 的已有 API Key

### 7) OAuth 授权状态动态化

- 授权状态中的剩余有效期改为前端 1s 级动态倒计时
- 倒计时归零后自动触发状态刷新，避免静态显示过期信息

### 8) 挂载决策稳定性增强

- `decide_mount_node` 增加网络抖动重试（最多 3 次）
- 覆盖典型瞬时错误：`incomplete chunked read` / `peer closed connection` / `read timeout` 等
- 重试仍失败时保持原有降级策略：挂载到根目录并记录日志
