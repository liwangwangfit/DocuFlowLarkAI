# 飞书 API 权限需求清单

## 问题说明

当前遇到两个主要问题：
1. **创建知识空间返回 400 错误** - 可能是应用未开通权限
2. **DOCX 上传转换问题** - 可能是转换器异常

---

## 需要的飞书权限

请在 [飞书开放平台](https://open.feishu.cn/app) 的"权限管理"中申请以下权限：

### 必需权限

| 权限名 | 权限代码 | 用途 | 紧急度 |
|--------|----------|------|--------|
| 创建知识空间 | `wiki:space:write` | 创建新的知识空间 | 🔴 高 |
| 查看知识空间 | `wiki:space:read` | 获取已有知识空间列表 | 🔴 高 |
| 创建文档 | `docx:document:write` | 在知识空间创建文档 | 🔴 高 |
| Markdown转换 | `docx:document.block:convert` | 将Markdown转为飞书Block | 🔴 高 |
| 查看文档 | `docx:document:read` | 读取文档内容 | 🟡 中 |
| 上传图片 | `drive:media:write` | 上传图片到文档 | 🟢 低 |
| 查看图片 | `drive:media:read` | 读取图片内容 | 🟢 低 |

---

## 手动创建知识空间（绕过 400 错误）

如果暂时无法解决权限问题，可以：

1. 在飞书客户端手动创建一个知识空间
2. 在系统配置的"知识空间 ID"字段填入该空间的 ID
3. 或者在任务创建后，使用"获取知识空间列表"功能选择已有空间

---

## 测试接口

我已经添加了降级逻辑：
- 创建空间失败时，会自动尝试获取已有空间列表
- 如果获取成功，会使用第一个已有空间
- 如果都失败，任务会报错并停止

---

## DOCX 转换问题

DOCX 转换使用 `python-docx` 库，请确保已安装：

```bash
pip install python-docx
```

如果转换失败，会返回基础文本内容作为降级方案。
